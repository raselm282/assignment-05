What is the Document Object Model(DOM)?

The Document Object Model (DOM) is a programming interface for representing HTML, XML, and other structured documents as a tree structure. It provides a way for JavaScript to interact with the content, structure, and style of a document.

What is event delegation in the context of the DOM, and why is it useful?

Event Delegation in the DOM refers to the practice of attaching an event listener to a parent element instead of directly to each child element. This allows you to handle events for multiple child elements efficiently, especially when the number of child elements is dynamic or large.

Why is it useful?

1. Improved Performance: When you attach event listeners directly to each child element, the browser has to create and manage a separate event listener object for each element. This can lead to performance issues, especially for large numbers of elements. With event delegation, you only need one event listener on the parent element, which is more efficient.

2. Dynamic Content: If your content is generated dynamically (e.g., using JavaScript), adding event listeners to each newly created element can be cumbersome. Event delegation provides a more flexible approach, as you only need to update the event listener on the parent element.

3. Simplified Code: Event delegation can lead to cleaner and more maintainable code, as you only need to write one event listener instead of multiple ones.

How do you manipulate an element's attributes and styles using the DOM?

Manipulating Element Attributes and Styles in the DOM Accessing Attributes

To access or modify an element's attributes, you can use the following methods:

1. getAttribute(attributeName): Returns the value of the specified attribute.

2. setAttribute(attributeName, value): Sets the value of the specified attribute.

3. removeAttribute(attributeName): Removes the specified attribute.
